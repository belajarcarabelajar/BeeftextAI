# Beeftext AI Deployment & Auto-Updater Guide

## 1. Preparation

**CRITICAL: Synchronize the application version** everywhere before building a new release. 
Increase the version number identically in:
1. `package.json`
2. `src-tauri/Cargo.toml`
3. `src-tauri/tauri.conf.json`

*(Note: The React frontend in `src/App.tsx` is now configured to fetch the version dynamically from Tauri using `getVersion()`, so you do not need to manually update the version string in the UI anymore.)*

## 3. Generate Release Build

Run the build commands:

```powershell
npm run build
npm run tauri build
```

*(Ensure `createUpdaterArtifacts: true` is properly set inside the `bundle` object in `tauri.conf.json`).*

## 4. GitHub Release

Upload the raw `.exe`, `.msi`, and their `.sig` files. Tauri v2 NSIS auto-updaters use the raw `.exe` (not `.exe.zip`).

```powershell
gh release create v0.2.0 `
  --title "BeefText AI v0.2.0" `
  --notes "Release notes" `
  "src-tauri\target\release\bundle\nsis\BeefText AI_0.2.0_x64-setup.exe" `
  "src-tauri\target\release\bundle\nsis\BeefText AI_0.2.0_x64-setup.exe.sig" `
  "src-tauri\target\release\bundle\msi\BeefText AI_0.2.0_x64_en-US.msi" `
  "src-tauri\target\release\bundle\msi\BeefText AI_0.2.0_x64_en-US.msi.sig"
```

*Warning: GitHub CLI (`gh`) replaces spaces with dots in the uploaded filenames (e.g., `BeefText.AI_0.2.0_x64-setup.exe`).*

## 5. Maintenance (latest.json and Github Upload)

1. Read the exact `.sig` string directly from the file (e.g., using `cat` or opening in VSCode to copy).
1. Update `latest.json` in the project root. **CRITICAL:** Ensure the `url` explicitly matches the remote GitHub asset name (with dots if GitHub renamed it, e.g. `BeefText.AI_0.2.0_x64-setup.exe`), **NOT** `.zip`.
1. Overwrite `latest.json` on the GitHub release:

   ```powershell
   gh release upload v0.2.0 latest.json --clobber
   ```

1. Commit and push `latest.json` to the `<master>` branch.
